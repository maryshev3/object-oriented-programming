# Обработка ошибок

## Введение

После знакомства с инкапсуляцией мы можем проектировать классы с контролируемым интерфейсом и скрытым состоянием. Однако надёжная программа должна не только защищать данные, но и уметь корректно реагировать на нештатные ситуации. В этой лекции мы разберём, как C++ позволяет обнаруживать и обрабатывать ошибки с помощью механизма исключений. Мы начнём с простого деления на ноль, познакомимся со стандартными типами исключений, научимся перехватывать их через ***try/catch***, а также создавать собственные классы исключений.

Представим функцию, выполняющую деление двух чисел, введённых пользователем:

```cpp
#include <iostream>

double divide(double a, double b) {
    return a / b;
}

int main() {
    double x, y;
    std::cout << "Введите числитель и знаменатель: ";
    std::cin >> x >> y;
    double result = divide(x, y);
    std::cout << "Результат: " << result << std::endl;
    return 0;
}
```

В функцию *devide* в качестве знаменателя может быть передан 0, что является логической ошибкой, которую необходимо обрабатывать.

**Исключение** — это объект, содержащий сведения о возникшей проблеме. Когда возникает ошибка, мы можем сгенерировать (выбросить) исключение с помощью ключевого слова ***throw***. Выброшенное исключение прерывает обычный ход программы и передаётся той части кода, которая способна его обработать.

Модифицируем функцию *divide*, чтобы она проверяла знаменатель и в случае нуля выбрасывала исключение:

```cpp
double divide(double a, double b) {
    if (b == 0.0) {
        throw std::invalid_argument("Был передан параметр b = 0. Параметр должен быть отличным от 0.");
    }
    return a / b;
}
```

Теперь при *b == 0.0* выполнение *divide* немедленно прекращается, и управление переходит к обработчику исключений в вызывающей функции. Если такого обработчика нет, программа аварийно завершается.

## Стандартные типы исключений

C++ предоставляет семейство готовых классов исключений в заголовочном файле ***<stdexcept>***. Все они наследуются от ***std::exception*** и предназначены для разных ситуаций:

- ***runtime_error***: общий тип исключений, которые возникают во время выполнения;

- ***range_error***: исключение, которое возникает, когда полученный результат превосходит допустимый диапазон;

- ***overflow_error***: исключение, которое возникает, если полученный результат превышает допустимый диапазон;

- ***underflow_error***: исключение, которое возникает, если полученный в вычислениях результат имеет недопустимое отрицательное значение (выход за нижнюю допустимую границу значений);

- ***logic_error***: исключение, которое возникает при наличии логических ошибок к коде программы;

- ***domain_error***: исключение, которое возникает, если для некоторого значения, передаваемого в функцию, не определен результат;

- ***invalid_argument***: исключение, которое возникает при передаче в функцию некорректного аргумента;

- ***length_error***: исключение, которое возникает при попытке создать объект большего размера, чем допустим для данного типа;

- ***out_of_range***: исключение, которое возникает при попытке доступа к элементам вне допустимого диапазона.

Приоритетно следует использовать именно стандартные типы. Они уже известны всем разработчикам, документированы и содержат метод *what()*, возвращающий описание ошибки.

## Конструкция try/catch

Для обработки потенциально опасного кода используется конструкция ***try/catch***:

- в блоке try размещается код, который может выбросить исключение;

- после него следуют один или несколько блоков ***catch***, каждый из которых "ловит" исключения определённого типа.

Обновим функцию *main*, в которую добавим обработку исключений:

```cpp
#include <iostream>
#include <stdexcept>

double divide(double a, double b) {
    if (b == 0.0) {
        throw std::invalid_argument("Был передан параметр b = 0. Параметр должен быть отличным от 0.");
    }
    return a / b;
}

int main() {
    double x, y;
    std::cout << "Введите числитель и знаменатель: ";
    std::cin >> x >> y;
    
    try {
        double result = divide(x, y);
        std::cout << "Результат: " << result << std::endl;
    } catch (const std::invalid_argument& e) {
        std::cerr << "Ошибка: " << e.what() << std::endl;
    }
    return 0;
}
```

Теперь программа не "падает" — она выводит понятное сообщение и завершается штатно.

### Несколько блоков catch

При наличии нескольких блоков ***catch*** исключение обрабатывается первым подходящим по типу. Порядок следования блоков критически важен: более специфичные (производные) типы должны стоять раньше, чем более общие (базовые). Иначе более общий блок перехватит исключение до того, как оно дойдёт до специализированного обработчика.

Рассмотрим классическую иерархию: ***std::exception*** является базовым классом для всех стандартных исключений, в том числе для ***std::invalid_argument***. Это означает, что ***catch*** (***std::exception& e***) перехватывает все исключения, производные от ***std::exception***, включая ***std::runtime_error***, ***std::logic_error*** и пользовательские типы, унаследованные от него.

Пример с несколькими блоками ***catch***:

```cpp
try {
    // некоторый код, бросающий различные исключения
} catch (const std::out_of_range& e) {
    // обработка выхода за границы
} catch (const std::runtime_error& e) {
    // обработка общих ошибок времени выполнения,
    // но НЕ out_of_range (он уже перехвачен выше)
} catch (const std::exception& e) {
    // перехват всех остальных стандартных исключений
    // но НЕ out_of_range, runtime_error (они уже перехвачены выше)
} catch (...) {
    // перехват вообще всех исключений любого типа
    // но НЕ out_of_range, runtime_error, exception (они уже перехвачены выше)
}
```

Если поменять местами, например, ***std::exception*** и ***std::runtime_error***, то блок ***std::exception*** всегда будет срабатывать для ***std::runtime_error*** и до более конкретного обработчика дело не дойдёт. Компилятор может выдать предупреждение о недостижимом коде.

### Перехват любых исключений

Блок ***catch (...)*** предназначен для перехвата исключений любого типа. Он ставится последним в цепочке catch-блоков, чтобы гарантировать, что ни одно исключение не пройдёт незамеченным. Внутри такого блока у вас нет доступа к объекту исключения — он лишь констатирует факт ошибки.

Обычно в catch (...) выполняют минимальную очистку и либо пробрасывают исключение дальше, либо завершают программу, так как точная причина неизвестна.

## Рекомендации по обработке исключений

Блок ***catch*** — это место, где программа фиксирует ошибку и выполняет необходимые восстановительные действия. Чаще всего в нём:

- ***логируют ошибку***: записывают детали в файл или консоль для последующего анализа;

- ***закрывают системные объекты*** — сетевые подключения, дескрипторы файлов, соединения с базой данных;

- ***высвобождают ресурсы***, выделенные в блоке try: динамическую память, мьютексы и т.п.

По умолчанию после завершения блока ***catch*** выполнение программы продолжается. Однако бывают ситуации, когда после высвобождения ресурсов необходимо пробросить исключение вверх по стеку вызовов.

Например, пользователь запросил данные с сервера. Для запроса используется отдельный сервис, который открывает соединение с базой данных и через него получает записи. Частая ошибка в таких сценариях — тайм-аут: запрос не укладывается в отведённое время и прерывается. В блоке catch следует закрыть соединение, но чтобы оповестить пользователя о возникшей проблеме, нужно передать исключение вызывающей функции, которая и отобразит сообщение об ошибке.

```cpp
#include <iostream>
#include <stdexcept>
#include <string>

// Класс исключения для ошибки тайм-аута.
class TimeoutException : public std::runtime_error {
    public:
        TimeoutException(const std::string& msg) : std::runtime_error(msg) {}
};

// Имитация подключения к базе данных.
class DatabaseConnection {
    public:
        void open() {
            std::cout << "[БД] Соединение открыто." << std::endl;
        }
        void close() {
            std::cout << "[БД] Соединение закрыто." << std::endl;
        }
};

// Функция, имитирующая получение данных с сервера через БД.
std::string fetchData() {
    // Захватываем ресурсы (динамическое выделение).
    DatabaseConnection* db = new DatabaseConnection();
    db->open();

    // Дополнительно выделяем буфер для данных.
    char* buffer = new char[1024];

    try {
        // Эмуляция долгого выполнения запроса;
        // Условие, приводящее к тайм-ауту.
        bool timeout = true; // представим, что здесь реальная проверка
        if (timeout) {
            throw TimeoutException("Запрос к БД превысил лимит времени.");
        }

        // В нормальной ситуации данные копируются в buffer, затем обрабатываются
        // ...
        std::string result = "Данные из БД";
        
        // Освобождение ресурсов при успешном выполнении.
        delete[] buffer;
        db->close();
        delete db;
        return result;
    } catch (...) {
        // Безопасно высвобождаем все ресурсы.
        delete[] buffer;
        db->close();
        delete db;

        std::cout << "[Сервис] Ресурсы освобождены, проброс исключения." << std::endl;
        throw;  // перебрасываем то же исключение выше
    }
}

int main() {
    try {
        std::string data = fetchData();
        std::cout << "Пользователь получил: " << data << std::endl;
    } catch (const TimeoutException& e) {
        std::cerr << "Ошибка (пользователю): " << e.what() << std::endl;
    } catch (const std::exception& e) {
        std::cerr << "Стандартная ошибка: " << e.what() << std::endl;
    } catch (...) {
        std::cerr << "Неизвестная ошибка." << std::endl;
    }
    return 0;
}
```

<div class="task-box"> <p><strong>Проброс исключения (rethrow)</strong> — ключевое слово <code>throw;</code> внутри блока <code>catch</code> без операнда перевыбрасывает то же самое исключение, сохраняя его тип и контекст. Это позволяет выполнить локальную очистку и передать ответственность вышестоящему обработчику.</p> </div>

## Пользовательские типы исключений

Иногда встроенных типов недостаточно. Например, вы пишете банковскую систему, и вам нужен тип ***InsufficientFundsException*** для ситуации нехватки средств. Создать его очень просто — достаточно унаследоваться от ***std::exception*** и переопределить метод *what()*:

```cpp
#include <iostream>
#include <stdexcept>
#include <string>

class InsufficientFundsException : public std::exception {
    public:
        InsufficientFundsException(const std::string& msg)
            : m_message(msg) {}

        const char* what() override {
            return m_message.c_str();
        }
    private:
        std::string m_message;
};

class BankAccount {
    public:
        BankAccount(double initial_balance)
            : m_balance(initial_balance) {}

        void withdraw(double amount) {
            if (amount > m_balance) {
                throw InsufficientFundsException(
                    "Недостаточно средств: запрошено " +
                    std::to_string(amount) +
                    ", доступно " + std::to_string(m_balance)
                );
            }
            m_balance -= amount;
        }

    private:
        double m_balance;
};
```

Теперь любая вызывающая сторона может ловить конкретно наше исключение, а также стандартное ***std::exception***:

```cpp
int main() {
    BankAccount account(1000.0);
    try {
        account.withdraw(2000.0);
    } catch (InsufficientFundsException& e) {
        std::cerr << "Банковская ошибка: " << e.what() << std::endl;
        // можно дополнительно инициировать бизнес-логику: уведомить клиента и т.д.
    } catch (std::exception& e) {
        std::cerr << "Произошла стандартная ошибка: " << e.what() << std::endl;
    }
    return 0;
}
```
