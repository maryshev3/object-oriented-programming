# Кейс ConsolePaint

## Задание

Компания разрабатывает терминальную утилиту для системных администраторов и DevOps-инженеров. Утилита работает на удалённых серверах по SSH и не имеет графического интерфейса. В отчётах, логах и дашбордах необходимо выводить простые ASCII-схемы (блоки, индикаторы, стрелки, геометрические фигуры).

Сейчас код отрисовки написан процедурно: каждая фигура обрабатывается отдельным *if/else* в ядре рендеринга. При добавлении каждой новой фигуры разработчики вынуждены лезть в ядро, править условия и рискуя сломать существующую логику. Тестирование занимает всё больше времени, а код становится неподдерживаемым.

Архитектор команды принял решение: выделить общий интерфейс фигуры и переписать ядро рендеринга на основе полиморфизма. Теперь добавление новой фигуры должно требовать создания только одного нового класса, без изменения кода отрисовки и меню.

Необходимо разработать консольное приложение **ConsolePaint**, которое:

- предлагает пользователю выбрать фигуру из списка (минимум 3 типа);

- запрашивает параметры фигуры (размер, символ отрисовки и т.п.);

- создаёт объект выбранной фигуры;

- отображает фигуру в терминале в виде ASCII-графики;

- позволяет повторить выбор или завершить работу.

Приложение должно быть спроектировано так, чтобы в будущем команда могла добавить новые фигуры (звёзды, стрелки, шестиугольники), не изменяя ни строку кода в функции рендеринга и в *main()*.
