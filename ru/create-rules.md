# Разработка RULES.md

## Задание

В рамках курса вы изучили ключевые принципы и технические приёмы объектно-ориентированного программирования на C++. Теперь вам предстоит выступить в роли разработчика стандартов кодирования для AI-ассистента (например, Qwen Code, GitHub Copilot и т.п.).

Ваша итоговая работа — файл **RULES.md**, содержащий правила и рекомендации, следуя которым нейросеть будет выдавать хорошо спроектированный, безопасный и сопровождаемый объектно-ориентированный код на C++.

Файл **RULES.md** должен быть написан на русском (или английском — по вашему выбору) языке и разбит на смысловые разделы. Каждое правило должно быть сформулировано ясно, лаконично и опираться на материалы изученных лекций. Где это уместно, приводите короткие примеры "правильного" и "неправильного" кода.

## Критерии оценивания

| **Критерий** | **Макс. балл** | **Комментарий** |
| --- | --- | --- |
| Полнота охвата | 3 | Правила покрывают все основные темы |
| Корректность и соответствие лекциям | 2 | Все правила верны технически и идеологически, не противоречат материалам курса |
| Практическая применимость | 2 | Правила сформулированы так, что ими может непосредственно руководствоваться AI-ассистент (чёткие инструкции, избегание неоднозначностей) |
| Структурированность и читаемость | 3 | 	Логичное деление на разделы, понятные заголовки, единообразное форматирование. Присутствуют примеры "хорошего" и "плохого" кода |
| ***Итого*** | ***10*** | |
